The Chrome Web Store listing is currently **in review**. Until it is approved, install the extension **unpacked** from the latest GitHub release. This is a one-time setup for developers and sales engineers.

## Prerequisites

- [Google Chrome](https://www.google.com/chrome/) (or a Chromium-based browser)
- The [xcsh CLI](https://f5xc-salesdemos.github.io/xcsh/) installed

## Install the extension (unpacked)

<Steps>

1. **Download the latest release.** Go to the [Releases page](https://github.com/f5xc-salesdemos/xcsh-chrome-extension/releases/latest) and download **`xcsh-chrome-extension-unpacked.zip`** (the "Install (unpacked)" asset).

2. **Unzip it** to a stable location you won't delete — for example `~/xcsh-chrome-extension/`.

3. **Open the extensions page.** Navigate to `chrome://extensions` in Chrome.

4. **Enable Developer mode** using the toggle in the top-right corner.

5. **Load the extension.** Click **Load unpacked** and select the unzipped folder.

6. **Verify the extension ID.** The loaded extension must show the ID:
   ```
   khlalklompggpfnmeclpligmcbknkemg
   ```
   This fixed ID is what the native-messaging bridge expects. (The `unpacked` build pins it via a manifest `key` — that is why you download the `-unpacked.zip`, not the plain `.zip`.)

</Steps>

## Connect the native-messaging bridge

<Steps>

1. **Run the setup command** to register the native-messaging host:
   ```bash
   xcsh chrome setup
   ```

2. **Reload a console tab.** Open or reload a tab on your F5 Distributed Cloud console (`*.volterra.us`).

3. **Confirm the connection.** Right-click the extension icon → **Options** — the status should show **Connected to xcsh** (green).

</Steps>

## Updating

When a new release is published, download the new `xcsh-chrome-extension-unpacked.zip`, replace your unzipped folder's contents, and click the **reload** (↻) icon on the extension card in `chrome://extensions`.

## Switching to the Chrome Web Store

Once the Chrome Web Store listing is approved, you can install from the store instead — remove the unpacked version first, then install from the store and re-run `xcsh chrome setup`.
